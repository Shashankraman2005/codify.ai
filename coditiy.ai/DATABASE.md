# DATABASE.md — Database Schema Reference

---

## Overview

JOBSEEK uses **PostgreSQL 15** as its sole datastore for both application data and Quartz Scheduler job triggers. The schema is managed entirely by **Flyway** migrations and is never auto-generated by Hibernate (`ddl-auto: validate`).

### Migration Files

| File | Purpose |
|------|---------|
| `V1__Initial_Schema.sql` | Core application tables, indexes, constraints |
| `V2__Quartz_Schema.sql` | Quartz JDBCJobStore tables (`QRTZ_*`) |
| `V3__Schedule_Tables.sql` | Extended Quartz schedule metadata |

## Table Descriptions

### `users`

Stores registered user accounts.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `username` | VARCHAR(100) | NOT NULL, UNIQUE | Login username |
| `password_hash` | VARCHAR(255) | NOT NULL | BCrypt-hashed password |
| `email` | VARCHAR(255) | NOT NULL, UNIQUE | User email address |
| `created_at` | TIMESTAMP(3) | NOT NULL, DEFAULT NOW() | Record creation time |
| `updated_at` | TIMESTAMP(3) | NOT NULL, DEFAULT NOW() | Last update time |

---

### `organizations`

Top-level multi-tenancy unit.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `name` | VARCHAR(100) | NOT NULL | Display name |
| `created_at` | TIMESTAMP(3) | NOT NULL | Creation time |
| `updated_at` | TIMESTAMP(3) | NOT NULL | Last update time |

---

### `organization_members`

Junction table between users and organizations. Stores the user's role within an org.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `organization_id` | BIGINT | FK → organizations(id) ON DELETE CASCADE | Parent org |
| `user_id` | BIGINT | FK → users(id) ON DELETE CASCADE | Member user |
| `role` | VARCHAR(50) | NOT NULL | `ADMIN`, `MEMBER`, or `VIEWER` |

**Unique constraint**: `(organization_id, user_id)` — one role per user per org.

---

### `projects`

A project lives within an organization and groups queues by application/service context.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `name` | VARCHAR(100) | NOT NULL | Project display name |
| `organization_id` | BIGINT | FK → organizations(id) ON DELETE RESTRICT | Parent organization |

**Delete restriction**: Cannot delete an org if it still has projects.

---

### `retry_policies`

Defines retry backoff behavior that queues can attach.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `name` | VARCHAR(100) | NOT NULL | Human-readable policy name |
| `strategy_type` | VARCHAR(50) | NOT NULL | `FIXED`, `LINEAR`, `EXPONENTIAL` |
| `base_delay_seconds` | INTEGER | NOT NULL | Base delay for backoff calculation |
| `max_delay_seconds` | INTEGER | NOT NULL | Upper cap on retry delay |
| `max_attempts` | INTEGER | NOT NULL | Jobs exceeding this are dead-lettered |

---

### `queues`

A queue is a named, prioritized, concurrency-limited channel for job execution within a project.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `name` | VARCHAR(100) | NOT NULL | Queue name (unique per project) |
| `project_id` | BIGINT | FK → projects(id) ON DELETE RESTRICT | Parent project |
| `priority` | INTEGER | NOT NULL, DEFAULT 0 | Queue-level priority (higher = processed first) |
| `concurrency_limit` | INTEGER | NOT NULL, DEFAULT 5 | Max parallel jobs per worker |
| `retry_policy_id` | BIGINT | FK → retry_policies(id) ON DELETE SET NULL, NULLABLE | Optional retry policy |
| `is_paused` | BOOLEAN | NOT NULL, DEFAULT FALSE | Stops workers from claiming new jobs |

**Unique constraint**: `(project_id, name)` — no duplicate queue names per project.
**Delete behavior**: `retry_policy_id` is SET NULL if the policy is deleted.

---

### `jobs`

The central table. Stores every job unit in its complete lifecycle.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `queue_id` | BIGINT | FK → queues(id) ON DELETE RESTRICT | Target queue |
| `parent_job_id` | BIGINT | FK → jobs(id) ON DELETE CASCADE, NULLABLE | For batch children |
| `type` | VARCHAR(50) | NOT NULL | `IMMEDIATE`, `DELAYED`, `SCHEDULED`, `RECURRING`, `BATCH` |
| `status` | VARCHAR(50) | NOT NULL | See Job Status values below |
| `payload` | TEXT | NULLABLE | JSON string payload passed to worker |
| `priority` | INTEGER | NOT NULL, DEFAULT 0 | Higher values are claimed first |
| `attempt_count` | INTEGER | NOT NULL, DEFAULT 0 | Number of times claimed |
| `max_attempts` | INTEGER | NOT NULL, DEFAULT 3 | Threshold for DLQ |
| `worker_id` | VARCHAR(255) | NULLABLE | ID of worker currently running/last ran this job |
| `scheduled_at` | TIMESTAMP(3) | NOT NULL | Earliest time a worker may claim this job |
| `started_at` | TIMESTAMP(3) | NULLABLE | When execution started |
| `completed_at` | TIMESTAMP(3) | NULLABLE | When execution ended |

**Job Status Values**: `QUEUED`, `CLAIMED`, `RUNNING`, `COMPLETED`, `FAILED`, `SCHEDULED`, `DEAD_LETTER`

---

### `job_executions`

Records each individual execution attempt for auditing.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `job_id` | BIGINT | FK → jobs(id) ON DELETE CASCADE | Parent job |
| `worker_id` | VARCHAR(255) | NOT NULL | Worker that ran this attempt |
| `attempt_number` | INTEGER | NOT NULL | Which attempt number this was |
| `started_at` | TIMESTAMP(3) | NOT NULL | Start of this execution |
| `ended_at` | TIMESTAMP(3) | NULLABLE | End of this execution |
| `status` | VARCHAR(50) | NOT NULL | `CLAIMED`, `RUNNING`, `COMPLETED`, `FAILED` |
| `error_message` | TEXT | NULLABLE | Error detail on failure |

---

### `job_logs`

Structured log entries emitted during job execution and system events.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `job_id` | BIGINT | FK → jobs(id) ON DELETE CASCADE | Parent job |
| `log_level` | VARCHAR(50) | NOT NULL | `INFO`, `WARN`, `ERROR` |
| `message` | TEXT | NOT NULL | Human-readable log message |
| `created_at` | TIMESTAMP(3) | NOT NULL | Log timestamp |

---

### `workers`

Registry of all worker nodes that have ever connected.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `worker_id` | VARCHAR(255) | PK | Unique worker identifier (env var or UUID) |
| `hostname` | VARCHAR(255) | NOT NULL | Docker container or machine hostname |
| `status` | VARCHAR(50) | NOT NULL | `ACTIVE`, `OFFLINE`, `DEAD` |
| `last_heartbeat_at` | TIMESTAMP(3) | NOT NULL | Most recent heartbeat time |
| `created_at` | TIMESTAMP(3) | NOT NULL | First registration time |

---

### `worker_heartbeats`

Historical heartbeat log per worker for diagnostics.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `worker_id` | VARCHAR(255) | FK → workers(worker_id) ON DELETE CASCADE | Parent worker |
| `last_heartbeat_at` | TIMESTAMP(3) | NOT NULL | Timestamp of this heartbeat |
| `status` | VARCHAR(50) | NOT NULL | Always `OK` |

---

### `dead_letter_queue`

Records jobs that have permanently failed after exhausting all retry attempts.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | BIGSERIAL | PK | Auto-increment primary key |
| `job_id` | BIGINT | FK → jobs(id) ON DELETE CASCADE, UNIQUE | The failed job |
| `final_error` | TEXT | NULLABLE | Last error message |
| `moved_at` | TIMESTAMP(3) | NOT NULL, DEFAULT NOW() | When job was quarantined |

**Unique constraint**: `job_id` — each job can only be in DLQ once.

---

### `scheduled_jobs`

Legacy cron-based scheduling table (used by `ScheduledJobService`). Quartz schedules (Phase 4+) are stored in the `schedules` table (V3 migration) and `QRTZ_*` tables.

---

### `locks`

Distributed lock table used for leader election (Reaper, and any future single-node operations).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `lock_name` | VARCHAR(100) | PK | Name of the lock |
| `holder_id` | VARCHAR(255) | NOT NULL | Worker ID holding the lock |
| `expires_at` | TIMESTAMP(3) | NOT NULL | TTL — lock is available if `expires_at < NOW()` |

## Indexes

| Index | Table | Columns | Purpose |
|-------|-------|---------|---------|
| `idx_jobs_queue_status_sched` | `jobs` | `(queue_id, status, scheduled_at)` | Primary worker polling index — covers the `SELECT FOR UPDATE SKIP LOCKED` query |
| `idx_jobs_status_sched` | `jobs` | `(status, scheduled_at)` | Reaper recovery and cross-queue status queries |
| `idx_worker_hb_wid_time` | `worker_heartbeats` | `(worker_id, last_heartbeat_at)` | Heartbeat history range queries |
| `idx_jobs_parent` | `jobs` | `(parent_job_id)` | Batch child lookup queries |

## Normalization

The schema is in **Third Normal Form (3NF)**:
- No repeating groups
- All non-key attributes depend on the primary key
- No transitive dependencies

`payload` is denormalized as a TEXT column (rather than a separate `job_payload` table) for simplicity and performance.

---

## Performance Notes

- The composite index `(queue_id, status, scheduled_at)` is the most critical index. It makes the worker polling query an index-only scan even on tables with millions of rows.
- `SKIP LOCKED` ensures that the index scan terminates immediately without visiting already-locked rows, making worker contention O(workers) rather than O(jobs).
- `TIMESTAMP(3)` avoids precision rounding when comparing heartbeat timestamps.
